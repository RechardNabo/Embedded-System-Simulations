🏛️ RECOMMENDED DEPLOYMENT TOPOLOGY
🔴 OT ZONE (Level 0–1)
• Field devices (PLC, BMS, sensors)
• Physical/serial bus isolation
• No direct internet access
• VLAN segmented from IT
• Protocol bridges in DMZ
🔓 Most protocols unencrypted here
🟡 DMZ / EDGE ZONE (Level 2)
• Protocol bridge containers
• MQTT broker (Mosquitto/EMQX)
• Firewall inbound :8883 only
• Certificates from internal CA
• No direct DB access
🔒 TLS required at this boundary
🔵 IT ZONE (Level 3)
• InfluxDB / TimescaleDB
• Grafana / SCADA server
• Node-RED automation
• Internal CA cert mgmt
• SIEM log collection
🔒 TLS + auth everywhere
🟢 CLOUD / REMOTE (Level 4)
• Broker bridge → AWS IoT Core
• Remote Grafana (HTTPS only)
• Mobile app (JWT + WSS)
• VPN for admin access
• Zero-trust access policy
🔒 mTLS + zero-trust required
✅ SECURITY BEST PRACTICES

🔒 TLS Everywhere

Use TLS 1.3 on all MQTT connections. Disable :1883 in production. Use Let's Encrypt or internal CA for broker certs.

🎫 mTLS for Bridges

Issue each bridge service a unique X.509 client certificate. Broker validates client certs — prevents rogue publishers.

📋 ACL Per Service

InfluxDB should only SUBSCRIBE to its topics. Bridges should only PUBLISH. Enforce read/write ACLs per client ID.

🔄 Key Rotation

Rotate MQTT passwords and API tokens every 90 days. Use short-lived JWT tokens (15 min) for browser clients via WSS.

🛡️ Network Segmentation

OT devices on isolated VLAN. Broker in DMZ. Only port 8883 exposed. Never expose :1883 or databases to the internet.

📊 Audit & Monitor

Subscribe to $SYS/# for broker metrics. Feed logs into SIEM. Alert on unexpected client connects or topic violations.

// WHY THIS ARCHITECTURE WINS

❌ WITHOUT MQTT HUB

  • 25 point-to-point integrations (5 protocols × 5 consumers)
  • Every app must parse Modbus, BACnet, SNMP, Zigbee, CAN frames
  • Protocol change breaks every consumer simultaneously
  • No central audit — data lost on consumer crash
  • Industrial binary protocols not internet/cloud friendly
  • Scaling requires rewiring entire mesh of connections

✅ WITH MQTT AS DATA BUS

  • Only 5 bridges — consumers just subscribe to clean JSON topics
  • Add new consumers with zero knowledge of underlying protocols
  • Each bridge is an isolated, replaceable Docker microservice
  • QoS 1/2 guarantees no data loss on reconnect + persistent sessions
  • Runs over WebSocket port 9001 — browser and cloud native
  • Topic wildcards allow fine-grained pub/sub filtering

🔌 Protocol Isolation

Each bridge knows only one protocol. Consumer failures never affect field data collection.

📨 Pub/Sub Decoupling

Publishers and subscribers are fully independent. Add InfluxDB without touching a single bridge.

🔁 QoS Guarantee

MQTT QoS 0/1/2 ensures messages survive network drops. Persistent sessions buffer data while consumers reconnect.

⚡ 2-byte Overhead

Minimal fixed header — ideal for embedded systems and high-frequency BMS/sensor telemetry at scale.

🌐 Universal Transport

TCP:1883 + TLS:8883 + WebSocket:9001. Browsers, mobile apps, and cloud services all connect natively.

🏗️ Microservice Ready

Each bridge is a Docker container. Scale, update, or swap any bridge independently with zero consumer downtime.
