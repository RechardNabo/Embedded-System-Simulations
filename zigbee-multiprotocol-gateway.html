❓ DOES ZIGBEE REQUIRE A SPECIAL "ROUTER"?
NO — A Zigbee router is NOT a WiFi router. It's any mains-powered Zigbee device flashed with router firmware (smart plug, bulb, switch). It has no IP address, runs no TCP/IP, and simply relays 2.4 GHz RF packets to extend the mesh.

NETWORK INFO ▼
⬢ Mesh Routing
Zigbee routers form a self-healing mesh. If a path fails, packets automatically reroute. The coordinator uses AODV (Ad-hoc On-demand Distance Vector) routing. Up to 10 hops supported, enabling large-scale networks.
AODV Routing Self-Healing Max 10 Hops ≤65,000 nodes
⬡ Gateway A (Building)
Multi-protocol IoT gateway bridging Zigbee, BLE, Z-Wave, Modbus, BACnet, and SNMP to IP. Runs Linux with protocol translation engine. Publishes all field data as MQTT/TLS to cloud (AWS/Azure).
Zigbee Coordinator MQTT Bridge Protocol Translation AES-128
⦿ LoRa Gateway
8-channel LoRaWAN gateway using Semtech SX1301/1302 concentrator. Receives on 868/915 MHz (SF7–SF12 simultaneous). Bridges LoRaWAN end devices to IP network via ChirpStack/TTN. PoE powered.
868/915 MHz SF7-SF12 8 Channels LoRaWAN 1.0.3
⬡ Gateway C (Warehouse)
Industrial logistics gateway for RFID asset tracking, GPS vehicle monitoring, LoRa cold storage sensors, and warehouse automation. Cellular 4G/5G backhaul. Runs edge analytics for inventory management.
UHF RFID 868MHz GPS/GNSS Multi-band Cellular 4G/5G Edge Analytics
🔒 Security & Profiles
Zigbee uses AES-128 link & network keys. Standard Zigbee profiles: ZHA (Home Automation), ZLL (Light Link), ZSE (Smart Energy). Zigbee 3.0 unifies all profiles. LoRaWAN uses AES-128 AppSKey & NwkSKey.
ZHA Profile ZLL Profile ZSE Profile Zigbee 3.0
⚡ Power & Sleep
End devices sleep between polls to save battery. Routers must stay awake to relay. Sleep periods configurable 250ms–60s. Typical coin cell (CR2032) lasts 1–3 years. LoRa nodes: 5–10 years on Li-SOCl2.
